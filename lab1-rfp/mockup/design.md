## 1. 目的と設計思想

### 目的
ワトソン建設株式会社への初回提案において、刷新後システムの操作イメージを具体的に示すためのUIモックアップ。

### 中心的な設計思想
> **「初見3秒でやばい工事がわかること」**

通常のダッシュボードが陥りがちな「全部見せる」設計を避け、**危険な工事を自動的に先頭・前面に押し出す**レイアウトを基本とした。ユーザーがスクロールや検索をしなくても、画面を開いた瞬間に要注意工事が目に入る状態を設計の最上位ゴールとしている。

## 3. データ構造

### プロジェクトデータ（`PROJECTS` 配列）

```js
// 1工事1オブジェクト
{
  id:             string,   // "K-001"
  name:           string,   // "大阪南港 倉庫新築工事"
  branch:         string,   // "大阪支店"
  manager:        string,   // "田中 誠司（所長）"
  phase:          string,   // "施工中"
  start:          string,   // "2026-04-01" (ISO8601)
  end:            string,   // "2027-01-31"
  contractAmount: number,   // 受注金額（千円）

  breakdown: [              // 工種別明細（配列）
    {
      cat:    string,       // "土工"
      budget: number,       // 実行予算（千円）
      actual: number,       // 発生原価（千円）
      vendors: [            // 担当協力会社（複数可）
        {
          name: string,     // "山田建材(株)"
          paid: number      // 支払済金額（千円）
        }
      ]
    }
  ]
}
```

### 導出値（`compute()` 関数）

`PROJECTS` の生データから以下をリアルタイム計算。ハードコードは一切なし。

| 導出フィールド | 計算式 | 意味 |
|---|---|---|
| `totalBudget` | `Σ breakdown[].budget` | 実行予算合計 |
| `totalActual` | `Σ breakdown[].actual` | 発生原価合計 |
| `plannedRate` | `totalBudget / contractAmount` | 実行予算率（受注比） |
| `actualRate` | `totalActual / contractAmount` | 発生原価率（受注比） |
| `alertDiff` | `(actualRate − plannedRate) × 100` | 受注時想定との乖離（pt） |
| `status` | `alertDiff ≥ 10pt → danger` / `≥ 5pt → warn` / それ以外 → `ok` | ステータス判定 |

### アラート閾値定数

```js
const ALERT_THRESHOLD = 10; // pt（別紙 FR-03 基準）
```

この1行を変更するだけで全画面のアラート判定・バッジ・バナーが連動する。

| 分類 | 技術 | 採用理由 |
|---|---|---|
| **マークアップ** | HTML5 | — |
| **スタイル** | CSS（Custom Properties / Flexbox / Grid） | ライブラリ不要。変数で色管理。 |

## 5. 機能詳細

### 5-1. アラート判定ロジック（FR-03）

```
alertDiff = (発生原価 / 受注金額) − (実行予算 / 受注金額)  ×100  [pt]

alertDiff ≥ ALERT_THRESHOLD(10)    → status = "danger"（要注意・赤）
alertDiff ≥ ALERT_THRESHOLD − 5(5) → status = "warn"  （注意・橙）
alertDiff < 5                       → status = "ok"    （正常・緑）
```

**アラート連動箇所：**
1. NAV タブバッジ（赤丸数字）
2. 画面最上部アラートバナー（赤帯・工事名一覧）
3. 工事カードの左ボーダー色・背景色
4. 工事カード内の「要注意」バッジ
5. 工種別内訳タブの見出し部 `⚠ 閾値超過` 表示

### 5-2. 工事カードのソートロジック（FR-03）

```js
DATA = PROJECTS.map(compute).sort((a,b) => b.alertDiff - a.alertDiff);
```

`alertDiff`（受注時想定との乖離pt）の降順でソート固定。ユーザーが何もしなくても「最も危ない工事が上」の状態が保証される。

### 5-3. アコーディオン展開（Tab 1）

工事カードをクリックすると、工種別明細テーブル＋差異チャートをインライン展開。別画面への遷移なし。複数工事の展開状態は独立して保持。

### 5-4. SVG差異チャート（Tab 1 / Tab 2）

- 横棒グラフ。中心線（±0%）を基準に、左が予算内・右が予算超過
- 棒の長さは工種間の相対比で正規化（最大値 = 最大幅）
- 色は差異テーブルと同じルール（赤 / 橙 / 緑）で統一
- `viewBox` 設定で可変幅に対応

### 5-5. フィルター連動（Tab 3）

工事フィルター・工種フィルターの変更時に `renderVendorTable()` を呼び直し、集計値をリアルタイムに再計算。フィルター変更のたびに合計行も更新される。

| # | 制約・事項 | 対応方針 |
